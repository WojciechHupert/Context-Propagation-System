Context Propagation System

A continuity layer for projects where information can spread, mutate, be contained, return as consequence and be studied.

What CPS Enables

A social-memory layer for authored worlds, research-facing prototypes, and cultural installations.

CPS can power authored immersive environments where characters, agents, or entities remember, interpret, and socially propagate information over time. It is built for worlds where conversation has an afterlife: where what is said can become known, misremembered, retold, hidden, or challenged later.

This makes CPS useful beyond a single genre. It can support narrative worlds, research-facing prototypes, interactive installations, technical memory systems, and experimental tools for observing how information moves through a group.

Social memory means that information is not stored only as an isolated fact. It becomes part of a shared environment.

A memory may belong to one agent, be withheld from another, spread through a defined path, change through retelling, or later return as a consequence in conversation. CPS treats this as infrastructure: memory has origin, ownership, privacy, lineage, and effect.

Propagation is the movement of information from one agent to another. In CPS, this is not just copying text from one memory database to another.

The system can track where the information came from, who passed it on, how many steps it travelled, whether it should remain private, and whether a new version is a duplicate or a meaningful mutation. This turns information flow into something observable rather than hidden.

Persistent AI systems can become difficult to understand. If agents remember things, influence each other, and act later based on hidden context, the system can quickly become opaque.

CPS is built around the opposite principle. It aims to make continuity visible. The system should allow a human operator to inspect memory lineage, propagation depth, privacy boundaries, duplicate suppression, runtime state, and the path by which a fact became socially active.

CPS Studio is the inspection and control surface for the system. It is where the current state of the artificial social environment can be observed.

Instead of trusting that the simulation “probably worked,” CPS Studio is intended to show what memories exist, where they came from, how they propagated, and whether the system behaved according to its rules.

At the current stage, Lelit Distrikt acts as the conversation client and presentation layer, while CPS is being positioned as the authority over meaning, memory creation, action creation, propagation policy, lineage, and recall context.

In practical terms, the visual world can host the interaction, but CPS decides how conversational events become persistent social memory and how that memory later influences the system.

The project currently combines an Unreal Engine-based world, a CPS backend, a local database, runtime APIs, agent memory records, propagation logic, and CPS Studio as the inspection interface.

The technical direction is modular: the visual layer, memory layer, propagation logic, runtime control state, and inspection tools should remain separable enough that CPS can evolve beyond one specific demo world.

The strongest current use case is a bounded authored environment where a small number of agents can demonstrate memory propagation clearly.

The Social Life of Information

Memory in a community is shaped by transmission, distortion, and context.

Every community is shaped by what it remembers, what it forgets, what it repeats, and what it misunderstands. Information rarely moves as a perfect fact. It travels through personality, emotion, trust, fear, loyalty, authority, and context. It can become rumor, warning, myth, suspicion, consensus, or conflict.

CPS was created to explore this process inside artificial worlds. Instead of treating memory as a private note attached to one character, CPS treats memory as social infrastructure. A piece of information can have a source, a path, a confidence level, an emotional charge, a history of mutation, privacy boundaries, and a visible lineage.

The important question is not only what a character says in the moment. The deeper question is what the world remembers after something has been said.

Enter The District

Lelit Distrikt is the first visual endpoint built around CPS.

Lelit Distrikt gives CPS a stage. It is a bounded, authored district where memory and information flow can be experienced rather than merely described. It does not try to simulate every aspect of life. It focuses on the social afterlife of conversation: what is remembered, what is shared, what is distorted, what remains hidden, and what returns later.

This is where CPS becomes emotionally legible. The system may track memory, lineage, and propagation, but Lelit turns those structures into moments: a character who heard something indirectly, a rumor that changed tone, a secret that stayed private, a conversation that matters because it did not end when the dialogue box closed.

From Creative Prototype to Research Platform

A bounded prototype for studying how information behaves inside artificial social environments.

CPS begins as an instrument-like prototype, but its long-term direction is broader. As it matures, it can support collaboration with researchers, technical teams, and cultural institutions exploring how information spreads through artificial social environments: how rumors mutate, how trust changes transmission, how emotionally charged memories persist, how institutions shape belief, and how external events alter group behavior.

The goal is not to simulate everything. The goal is to create bounded, observable systems where specific social-memory dynamics can be introduced, traced, inspected, and understood.

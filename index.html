00. Origins

The Fates

In classical mythology, the Moirai (the Fates) controlled the metaphorical thread of life of every mortal from birth to death. Clotho spun the thread, Lachesis measured its length, and Atropos cut it.

This digital processing system adopts their namesake, operating on the contemporary urban traces of humanity. It weaves fragmented data into cohesive memory blocks, measures their societal impact, and determines their decay.

02. Cognitive Architecture

Dual-Layer Memory

The architecture enforces a strict separation between what the system knows and what the character recalls. Moirai remembers accurately — NPCs remember humanly.

LAYER_01

System Memory

CANONICAL

The canonical SQLite database. Accurate, structured, append-only, and never destructively edited. Contains the full truth and the complete source chain of how information was acquired.

  • IntegrityImmutable append-only
  • StorageSQLite3 shared DB
  • Tables11 canonical tables
  • AccessRuntime + Studio shared
LAYER_02

Character Recall

VOLATILE

Generated at retrieval time. Imperfect, selective, and emotional. Characters may forget, distort, or hesitate based on their current state and relationship to the information.

  • NatureActive filter, not stored data
  • BehaviorsForgetting, vagueness, drift
  • InfluenceAge, stability, emotion
  • OutputHumanized prompt context
// AI_RESPONSIBILITY

AI IS NOT the source of truth — it cannot create hidden state outside the database.

04. Cognition

The Recall Filter

A lightweight processing layer sits between the canonical database and the live chat prompt. It transforms perfect system records into imperfect, human-like NPC memory.

01

Retrieval Bucketing

Memory is categorized into four buckets — Recent, Stable, Social, and Emotional — each with hard token/count caps to prevent context sprawl.

02

Age & Stability Check

Each record's RecallStability (stable / degradable / fragile) and creation timestamp determine how clearly the NPC can access it. Old fragile memories blur.

03

Distortion Pass

Records with high DistortionRisk undergo phrasing mutation — local rumor rephrasing, hedged certainty, or emotional recoloring based on the NPC's personality.

04

Speech Policy Injection

The filtered context enters the prompt with narrative-safe labels: "Your Experiences" for direct memories, "Local Rumors & News" for hearsay — bypassing LLM safety refusals.

05. Social Mechanics

Memory Propagation

Information doesn't broadcast randomly. It spreads through social and semantic resonance — scored by relationship, relevance, and emotional weight.

RULE_01 Associative Routing

Memories prefer propagation between NPCs sharing a ShareGroupName. Secondary resonance uses tags and narrative network proximity as a fallback path.

RULE_02 Perspective Ownership

Propagation always creates a new, distinct MemoryRecord owned by the recipient. The original record is never copied — only reinterpreted through the recipient's lens.

RULE_03 Privacy Separation

Only memories with PrivacyLevel of "shareable" or "public" can propagate. Private memory never spreads automatically — privacy leaks must be designed as narrative events.

RULE_04 Cognitive Drift

Each propagation hop increments MutationCount and preserves OriginalSummaryText. Confidence degrades through recall. Hard cap: maximum 2 social hops.

RULE_05 Loop Suppression

Social actions are blocked for existing hearsay unless the target is directly implicated. Maximum 15 propagation events per synthesis cycle, 1 recipient per fact per cycle.

07. Intent Management

Action Lifecycle

NPC actions represent intent, not fixed dialogue. Each follows a strict state machine to prevent stale repetition and enforce temporal believability.

Pending

Intent extracted from conversation. Awaiting trigger window. Has a TriggerWindowStart defining earliest activation.

→

Triggered

Window opened. Action injected into NPC prompt context for the next relevant conversation encounter.

→

Used

Action surfaced in dialogue. Marked complete. Enters CooldownUntil period to prevent immediate repetition.

→

Expired

Window passed without triggering, or cooldown elapsed. Converted to past reflection. Never re-enters active prompt.

09. Constraints

Risk Analysis

Every design decision in Moirai addresses a specific failure mode. The system is built around mitigating these behavioral risks.

Over-Sharing

Town feels too small. NPCs gossip about everything instantly.

FIX → Strict propagation eligibility. Capped events per synthesis cycle. Privacy separation.

Over-Precision

Robotic behavior. NPCs perfectly quote past conversations verbatim.

FIX → Recall filter introduces vagueness and hesitation based on RecallStability and elapsed time.

Over-Distortion

Continuity breaks. NPCs confidently claim things that never happened.

FIX → Core facts immutable in Moirai. Recall filter restricted from mutating stable, low-risk memories.

Repetition

NPCs bring up the same hook every conversation. Breaks immersion.

FIX → Action lifecycle enforcement. Triggered/used actions never re-enter future prompts.
